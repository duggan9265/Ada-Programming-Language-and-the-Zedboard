\section{Vitis: Setting up the Software Side}

Vivado is used for the \gls{pl}-side of Zynq, and Vitis for the \gls{ps}-side. To setup properly:

\begin{enumerate}
    \item Create a folder inside c:\, or somewhere with a short pathname called workspace. If any files exceed the Windows limit of 256, the platform will not build.
    \item Open Vitis. Select Open Workspace, and open the workspace you just created.
    \item Select File -> New Component -> Platform.
    \item Give it a name (default is fine), ensure it will live inside workspace.
    \item Select next. Select Hardware Design and browse for the .tcl file inside the temp folder.
    \item Select next. Use Operating System standalone and the default processor. 
    \item Select Generate Boot Artifacts
    
    \begin{center}
        \includegraphics[width=0.9\textwidth]{aa_latex/Figures/Blink_LED/Vitis/vitis.png}
    \end{center}

    \item Select Next and Finish.
    \item Vitis should build the platform.
    
     \begin{center}
        \includegraphics[width=0.9\textwidth]{aa_latex/Figures/Blink_LED/Vitis/platform.png}
    \end{center}
\end{enumerate}

Once this is done, the bitstream can be downloaded to the Zedboard by selecting Vitis->Program Device. Make the sure the JTAG cable is connected of course.

\begin{center}
        \includegraphics[width=0.9\textwidth]{aa_latex/Figures/Blink_LED/Vitis/prog_device.png}
    \end{center}

We now need to download the main.elf file to run on the processing system. I did this using the Xilinx Software Command-Line Tool (XSCT). In the terminal do the following


\begin{enumerate}
    \item Open a terminal and start XSCT:
\begin{lstlisting}[style=xdc]
xsct
\end{lstlisting}

    \item Connect to the target board via JTAG:
\begin{lstlisting}[style=xdc]
connect
\end{lstlisting}

    \item List available targets:
\begin{lstlisting}[style=xdc]
targets
\end{lstlisting}

    You should see output similar to:
\begin{lstlisting}
1  APU
   2  ARM Cortex-A9 MPCore #0 (Vector Catch)
   3  ARM Cortex-A9 MPCore #1 (Running)
4  xc7z020
\end{lstlisting}

    \item Select the first ARM core (CPU 0):
\begin{lstlisting}[style=xdc]
targets 2
\end{lstlisting}

    \item Reset the processor to ensure a clean state:
\begin{lstlisting}[style=xdc]
rst -processor
\end{lstlisting}

    \item Download the compiled Ada executable (\texttt{.elf}) to memory:
\begin{lstlisting}[style=xdc]
dow "C:/path/to/your/project/bin/main.elf"
\end{lstlisting}

    \item Start execution:
\begin{lstlisting}[style=xdc]
con
\end{lstlisting}
\end{enumerate}

Once the program is running:
\begin{itemize}
    \item The ARM Cortex-A9 begins executing the Ada application
    \item The \texttt{Main} procedure is invoked via the GNAT runtime
    \item The LEDs connected via EMIO (LD1, LD3, LD5, LD7) should begin blinking according to the implemented logic
\end{itemize}
